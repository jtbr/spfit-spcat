# SPFIT/SPCAT Test Suite

This directory contains a collection of test cases for the SPFIT and SPCAT programs,
based on examples from the [CDMS Pickett software examples page](https://cdms.astro.uni-koeln.de/classic/predictions/pickett/beispiele/).

## Purpose

The purpose of this suite is to:
1. Provide a standardized set of inputs for testing SPFIT and SPCAT installations.
2. Allow verification of results by comparing generated outputs against reference outputs.

## Directory Structure

The test suite is organized as follows:

spfit_spcat_test_suite/
├── README.md
├── [category_name]/ # e.g., diatomic_molecules
│ ├── [molecule_example_name]/ # e.g., CO
│ │ ├── [basename].par # Input parameter file for SPFIT
│ │ ├── [basename].lin # Input line file for SPFIT
│ │ ├── [basename].int # Input intensity file for SPCAT
│ │ └── reference_outputs/
│ │ ├── [basename].fit # Reference output from SPFIT
│ │ ├── [basename].cat # Reference output from SPCAT, if available
│ │ └── [basename].out # Reference general output from SPCAT, if available
│ │ └── v2008_outputs/
│ │ ├── [basename].fit # Reference output from SPFIT
│ │ ├── [basename].cat # Reference output from SPCAT
│ │ └── [basename].out # Reference general output from SPCAT

## Running Tests

**Prerequisites:**
* SPFIT and SPCAT programs installed and accessible in your system's PATH.

**General Procedure for each test case:**

1.  **Navigate to the example directory:**
    ```bash
    cd spfit_spcat_test_suite/[category_name]/[molecule_example_name]/
    ```

2.  **Run SPFIT:**
    SPFIT uses the `.par` and `.lin` files. It will generate a `.fit` file (containing the results of the fit) and a `.var` file (containing the fitted parameters, used by SPCAT), among others.
    ```bash
    spfit [basename]
    ```
    For example, in the `CO` directory: `spfit co`

3.  **Run SPCAT:**
    SPCAT typically uses the `.var` file (generated by SPFIT) and an `.int` file. It generates a `.cat` file (catalog output) and often a `.out` file (general information).
    ```bash
    spcat [basename]
    ```
    For example, in the `CO` directory: `spcat co`
    (This assumes `[basename].var` was created by `spfit [basename]` in the same directory).

## Verifying Results

After running SPFIT and SPCAT, compare the generated output files (e.g., `[basename].fit`, `[basename].cat`) with the files in the `reference_outputs/` subdirectory.

*   **Text-based comparison (for `.fit`, `.out`):**
    ```bash
    diff [basename].fit reference_outputs/[basename].fit
    diff [basename].out reference_outputs/[basename].out
    ```
*   **Catalog comparison (`.cat` files):**
    `.cat` files contain numerical data. Small differences due to floating-point arithmetic or minor program version changes are possible. A simple `diff` might show many differences. More sophisticated comparison might be needed for automated checks, possibly by comparing values within a certain tolerance.
    ```bash
    diff [basename].cat reference_outputs/[basename].cat
    ```

    For a more robust comparison of `.cat` files, consider using tools or scripts that can parse the format and compare numerical columns with a tolerance.

## Notes

*   SPFIT and SPCAT overwrite existing output files in the current directory. Each test case is in its own directory to prevent interference.
*   The reference output files were downloaded from the CDMS Pickett examples website.
